DRC warning 当前主要包括 `CFGBVS/CONFIG_VOLTAGE` 未显式设置；带 ILA 版本还包含 ILA/调试网络相关 warning，不阻塞 bitstream。若老师要求完全清 warning，需要根据板卡配置电压补充对应 XDC 属性。

## 6. 上板步骤

1. 打开 Vivado Hardware Manager。
2. 连接 K7EDAEVAL，执行 `Open Target -> Auto Connect`。
3. Program Device，选择 `board_top_ila.bit`。
4. 同时加载 `board_top_ila.ltx`。
5. 打开 ILA，观察 `done`、`verify_we`、`verify_addr`、`verify_vector_out` 和 `cnt_test`。

判断标准：

| 信号 | 期望 |
| --- | --- |
| `done` | 最终拉高 |
| `verify_we` | 出现 16 次有效写入 |
| `verify_addr` | 最后写地址为 15 |
| `cnt_test` | 接近 157 |
| `verify_vector_out` | 写出顺序与 `mem/FFT_output.coe` 一致 |

LED 映射：

| LED | 信号 |
| --- | --- |
| LED1 | `done` |
| LED2 | `verify_we` |
| LED3 到 LED7 | `cnt_test` 部分位 |
| LED8 | `verify_RAM` debug 数据异或值 |

ILA probe：

| probe | 信号 |
| --- | --- |
| `probe0` | `test_vector_in[15:0]` |
| `probe1` | `verify_vector_out[15:0]` |
| `probe2` | `verify_we` |
| `probe3` | `verify_addr[4:0]` |
| `probe4` | `cnt_test[19:0]` |
| `probe5` | `done` |

## 8. 时钟说明

当前 `rtl/board_top.v` 直接使用板载 50 MHz：

```verilog
assign clk = CLK_50M;
```

因此下载到板上后的真实运行频率是 50 MHz。路线矩阵中的 95/100/110/120/130 MHz 是 Vivado 时序目标，用于判断 RTL 的收敛余量，不会自动生成真实高频时钟。如果需要板上实际高频运行，需要加入 PLL/MMCM 或外部高频时钟，并同步修改 `board_top.v` 和 XDC。

## 9. 后续可选优化

1. 核对板卡配置电压后，在 XDC 中补充 `CFGBVS` 和 `CONFIG_VOLTAGE`。
2. 用 `board_utilization_hierarchical_no_ila.rpt` 作为资源统计依据，记录 `u_mcu_top` 而不是整个测试平台。
3. 若要继续追求频率，可运行 `routes/speed_v8_route_a_vivado_matrix` 的矩阵脚本比较 v6/v7/v7b/v7c。
4. 若常数 91 专用路线没有明显时序优势，建议保留 `speed_v7_q7_narrow_mul`，因为它语义更通用、风险更低。

## 10. 下一步操作顺序

建议按下面顺序推进，不要直接跳到无 ILA 版本：

1. 首次上板使用 `board_top_ila.bit` 和 `board_top_ila.ltx`，确认 ILA 中 `done` 拉高、`verify_we` 有 16 次有效写入、`verify_addr` 最后到 15。
2. 对照 `mem/FFT_output.coe` 检查 `verify_vector_out` 的 16 个输出值；若输出不一致，先保留带 ILA 版本继续抓波形。
3. 带 ILA 版本确认功能无误后，再切换到 `board_top_no_ila.bit` 做正式资源、速度和演示记录。
4. 正式资源表使用 `board_utilization_hierarchical_no_ila.rpt`，优先记录 `u_mcu_top` 层级，不把 `test_ROM`、`verify_RAM`、ILA 调试平台算入 MCU 本体资源。
5. 路线选择参考 `routes/speed_v8_route_a_vivado_matrix/results/leaderboard_summary.md`：若强调效率，优先看 `speed_v7c_c91_shift_sub`；若强调通用性和风险，保留 `speed_v7_q7_narrow_mul`。

关闭 ILA 不会改变 MCU 计算逻辑，只会移除在线波形抓取能力。调试阶段用带 ILA，提交资源和速度成绩时用无 ILA。
