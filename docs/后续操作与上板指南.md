## 6. 最终实现矩阵

补齐 `xc7k325tffg900-2` 后运行最终矩阵：

```powershell
subst M: C:\Users\戎择辰\OneDrive\文档\数电实验\MCU
cd M:\routes\speed_v8_route_a_vivado_matrix
D:\vivado\2025.2\Vivado\bin\vivado.bat -mode batch -source vivado\run_route_a_matrix.tcl
py scripts\parse_vivado_reports.py --root build\vivado_matrix --out results\route_a_matrix.csv
```

矩阵默认覆盖：

| 项目 | 默认值 |
| --- | --- |
| 路线 | v6、v7、v7b、v7c |
| 频率目标 | 95、100、110、120、130 MHz |
| Vivado strategy | `Performance_Explore`、`Performance_ExplorePostRoutePhysOpt` |
| 默认 part | `xc7k325tffg900-2` |

如果只想先跑推荐路线和低频目标，可以新建临时 wrapper：

```tcl
set PART_NAME xc7k325tffg900-2
set JOBS 4
set CANDIDATES {{speed_v7_q7_narrow_mul ../speed_v7_q7_narrow_mul/mcu_fft_q7_narrow_mul}}
set TARGETS {{95 10.526}}
set STRATEGIES {Performance_Explore}
source vivado/run_route_a_matrix.tcl
```

最终路线选择规则：

1. 先筛选 `WNS >= 0` 的结果。
2. 优先选择可收敛的最高频率。
3. 频率相同再比较 LUT、FF、DSP、BRAM。
4. 不把带 ILA 的资源计入最终成绩。
5. 若 v7、v7b、v7c 时序接近，优先选择语义更通用的 `speed_v7_q7_narrow_mul`。

## 8. 创建上板工程

进入推荐路线目录：

```powershell
subst M: C:\Users\戎择辰\OneDrive\文档\数电实验\MCU
cd M:\routes\speed_v7_q7_narrow_mul\mcu_fft_q7_narrow_mul
D:\vivado\2025.2\Vivado\bin\vivado.bat
```

Vivado Tcl Console：

```tcl
set PART_NAME xc7k325tffg900-2
set TARGET_PERIOD_NS 20.000
set ENABLE_ILA 1
source ../../vivado/create_board_project.tcl
```

说明：

| 参数 | 含义 |
| --- | --- |
| `PART_NAME` | K7EDAEVAL 实际 FPGA 型号。 |
| `TARGET_PERIOD_NS 20.000` | 50 MHz 板载时钟约束。 |
| `TARGET_PERIOD_NS 10.526` | 95 MHz 时序目标，只用于实现约束，不会自动产生真实 95 MHz 时钟。 |
| `ENABLE_ILA 1` | 首次调试打开 ILA。 |
| `ENABLE_ILA 0` | 最终资源和速度比较关闭 ILA。 |

综合、实现和生成 bitstream：

```tcl
launch_runs synth_1 -jobs 4
wait_on_run synth_1
launch_runs impl_1 -to_step write_bitstream -jobs 4
wait_on_run impl_1
open_run impl_1
report_timing_summary
report_utilization
```

若 `WNS < 0`，该目标频率不能作为最终速度成绩。

## 9. 时钟注意事项

当前 `rtl/board_top.v` 中：

```verilog
assign clk = CLK_50M;
```

这表示板上真实运行时钟仍然是 K7EDAEVAL 的 50 MHz 输入。95/100/110/120/130 MHz 矩阵只是 Vivado 时序目标，用来判断 RTL 能否在这些周期下收敛。

若验收要求板上真实运行 95 MHz 以上，需要额外加入 PLL/MMCM 或外部高频时钟，并同步修改：

```text
rtl/board_top.v
constraints/top.xdc
```

当前仓库没有固化 PLL/MMCM，目的是避免引入与具体 Vivado/IP 版本绑定的额外依赖。

## 11. LED 与 ILA 观察

板级 LED：

| 信号 | 含义 |
| --- | --- |
| `LED1` | `done`，程序完成标志。 |
| `LED2` | `verify_we`，输出 RAM 写使能。 |
| `LED3` 到 `LED7` | `cnt_test` 的部分位。 |
| `LED8` | `verify_RAM` debug 数据异或。 |

ILA probe：

| probe | 信号 |
| --- | --- |
| `probe0` | `test_vector_in[15:0]` |
| `probe1` | `verify_vector_out[15:0]` |
| `probe2` | `verify_we` |
| `probe3` | `verify_addr[4:0]` |
| `probe4` | `cnt_test[19:0]` |
| `probe5` | `done` |

首次上板重点确认：

1. `done` 最终拉高。
2. `verify_we` 出现 16 次有效写入。
3. 最后一次写地址为 15。
4. `cnt_test` 接近 157。
5. `verify_vector_out` 写入顺序与 `mem/FFT_output.coe` 一致。

## 12. 故障排查

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| Vivado 找不到 `xc7k325tffg900-2` | 未安装目标器件包或 license 不支持 | 在 Vivado installer/license manager 中补齐 Kintex-7 K325 FFG900 支持。 |
| `$readmemh` 找不到 `mem/*.mem` | 从错误工作目录启动 Vivado | 使用已更新脚本，或先 `cd` 到路线工程根目录。 |
| `.Xil/.../realtime` 无法删除 | Vivado 临时目录被锁 | 关闭 Vivado 进程，删除 `.Xil`，换新的 `OUT_DIR` 重跑。 |
| `done` 不拉高 | 复位、时钟、指令 ROM 或程序异常 | 检查 `KEY1` 极性、`CLK_50M`、`mem/instr_fft8.mem` 和 ILA 波形。 |
| `verify_we` 少于 16 次 | 程序未完整写出输出 | 回到仿真看 `verify_addr`、`cnt_test`、分支跳转。 |
| `WNS < 0` | 时序目标过高或关键路径过长 | 降频、换 strategy、比较 v7/v7b/v7c。 |
