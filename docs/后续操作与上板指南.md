矩阵默认覆盖：

| 项目 | 默认值 |
| --- | --- |
| 路线 | v6、v7、v7b、v7c |
| 频率目标 | 95、100、110、120、130 MHz |
| Vivado strategy | `Performance_Explore`、`Performance_ExplorePostRoutePhysOpt` |
| 默认 part | `xc7k325tffg900-2` |

如果只想先跑推荐路线和低频目标，可以新建临时 wrapper：

```tcl

set JOBS 4
set CANDIDATES {{speed_v7_q7_narrow_mul ../speed_v7_q7_narrow_mul/mcu_fft_q7_narrow_mul}}
set TARGETS {{95 10.526}}
set STRATEGIES {Performance_Explore}
source vivado/run_route_a_matrix.tcl
```

最终路线选择规则：

1. 先筛选 `WNS >= 0` 的结果。
2. 优先选择可收敛的最高频率。
3. 频率相同再比较 LUT、FF、DSP、BRAM。
4. 不把带 ILA 的资源计入最终成绩。
5. 若 v7、v7b、v7c 时序接近，优先选择语义更通用的 `speed_v7_q7_narrow_mul`。

## 7. 上板前完成度检查

不要在只完成综合冒烟后直接下载板卡。真正上板前至少要完成下面这些 gate：

| Gate | 必须完成的事情 | 通过标准 |
| --- | --- | --- |
| 环境 gate | Vivado 能识别 `xc7k325tffg900-2`，并且 license 可用于综合、实现和 bitstream。 | `get_parts xc7k325tffg900-2` 有结果；创建工程和实现不会因器件/license 失败。 |
| 路径 gate | 使用短路径运行 Vivado。 | 当前工作目录在 `M:\...`，没有中文路径、OneDrive 同步锁或超长路径干扰。 |
| 功能 gate | 跑推荐路线或全部路线的 Verilog 回归。 | `official_sample PASS`，随机样例 PASS；没有因为缺 `iverilog/vvp` 被误判。 |
| 初始化 gate | 确认上板用的 ROM/RAM 初始化文件是最新的。 | `mem/FFT_input.mem`、`mem/instr_fft8.mem`、`mem/instr_fft8.coe`、`mem/FFT_output.coe` 存在且来自刚跑过的回归。 |
| 约束 gate | 检查板级顶层和 XDC。 | `rtl/board_top.v`、`constraints/top.xdc` 中时钟、按键、LED 引脚与 K7EDAEVAL 实物一致。 |
| 实现 gate | 对候选路线完成目标 part 下的综合和实现。 | `synth_1`、`impl_1` 成功，目标频率下 `WNS >= 0`，无阻止 bitstream 的 DRC error。 |
| 调试 gate | 首次上板工程打开 ILA。 | 生成 `.bit` 和 `.ltx`，probe 与本文档中的信号映射一致。 |
| 记录 gate | 保存结果供队友复查。 | 保存 timing summary、utilization report、bit/ltx 路径、所选路线和频率目标。 |

建议上板前用下面的文件清单逐项确认：

| 文件 | 检查内容 |
| --- | --- |
| `rtl/board_top.v` | `CLK_50M` 是否仍作为主时钟；`KEY1` 复位极性是否符合按键实测；LED/ILA 信号是否接到预期内部信号。 |
| `constraints/top.xdc` | `create_clock` 周期、FPGA 管脚、IOSTANDARD 是否与实验板资料一致。 |
| `mem/instr_fft8.mem` | 指令 ROM 初始化文件存在，Vivado log 中没有 `$readmemh` 找不到文件。 |
| `mem/FFT_input.mem` | 输入数据已由最近一次回归生成。 |
| `mem/FFT_output.coe` | 用作 ILA 输出核对基准。 |
| `vivado/create_board_project.tcl` | `PART_NAME`、`TARGET_PERIOD_NS`、`ENABLE_ILA` 使用本次上板所需设置。 |

硬件连接也要在下载前检查：

| 项目 | 要求 |
| --- | --- |
| 板卡 | 确认为 K7EDAEVAL，FPGA 型号与工程 part 一致。 |
| 供电 | 电源适配器、电源开关、板载电源指示灯正常。 |
| JTAG | Vivado Hardware Manager 能 Auto Connect 并识别 FPGA。 |
| 时钟 | 板载 `CLK_50M` 可用；当前仓库未固化 PLL/MMCM。 |
| 复位 | 按下/松开 `KEY1` 后，ILA 中 `cnt_test` 能重新开始变化。 |
| 跳线/拨码 | 以实验板手册为准确认启动模式、JTAG 模式和电源选择；本文不硬写具体位置，避免和不同批次板卡冲突。 |

## 8. 推荐上板路线

首次上板建议使用：

```text
routes/speed_v7_q7_narrow_mul/mcu_fft_q7_narrow_mul
```

理由：

- 已通过官方样例和随机样例的既有回归记录。
- 综合冒烟 LUT 最低。

- 相比常数 91 专用路线，功能语义更通用。

## 9. 创建上板工程

进入推荐路线目录：

```powershell
subst M: C:\Users\戎择辰\OneDrive\文档\数电实验\MCU
cd M:\routes\speed_v7_q7_narrow_mul\mcu_fft_q7_narrow_mul
D:\vivado\2025.2\Vivado\bin\vivado.bat
```

Vivado Tcl Console：

```tcl
set PART_NAME xc7k325tffg900-2
set TARGET_PERIOD_NS 20.000
set ENABLE_ILA 1
source ../../vivado/create_board_project.tcl
```

说明：

| 参数 | 含义 |
| --- | --- |
| `PART_NAME` | K7EDAEVAL 实际 FPGA 型号。 |
| `TARGET_PERIOD_NS 20.000` | 50 MHz 板载时钟约束。 |
| `TARGET_PERIOD_NS 10.526` | 95 MHz 时序目标，只用于实现约束，不会自动产生真实 95 MHz 时钟。 |
| `ENABLE_ILA 1` | 首次调试打开 ILA。 |
| `ENABLE_ILA 0` | 最终资源和速度比较关闭 ILA。 |

综合、实现和生成 bitstream：

```tcl
launch_runs synth_1 -jobs 4
wait_on_run synth_1
launch_runs impl_1 -to_step write_bitstream -jobs 4
wait_on_run impl_1
open_run impl_1
report_timing_summary
report_utilization
```

若 `WNS < 0`，该目标频率不能作为最终速度成绩。

生成完成后，把以下结果记录下来：

| 结果 | 说明 |
| --- | --- |
| `.bit` 路径 | 下载 FPGA 必需。 |
| `.ltx` 路径 | 启用 ILA 时必须和 `.bit` 配套。 |
| timing summary | 重点记录 WNS、TNS、失败路径数量。 |
| utilization report | 重点记录 LUT、FF、DSP、BRAM。 |
| Vivado warnings | 尤其关注 DRC、未约束路径、时钟约束和 `$readmemh` 相关提示。 |

## 10. 时钟注意事项

当前 `rtl/board_top.v` 中：

```verilog
assign clk = CLK_50M;
```

这表示板上真实运行时钟仍然是 K7EDAEVAL 的 50 MHz 输入。95/100/110/120/130 MHz 矩阵只是 Vivado 时序目标，用来判断 RTL 能否在这些周期下收敛。

若验收要求板上真实运行 95 MHz 以上，需要额外加入 PLL/MMCM 或外部高频时钟，并同步修改：

```text
rtl/board_top.v
constraints/top.xdc
```

当前仓库没有固化 PLL/MMCM，目的是避免引入与具体 Vivado/IP 版本绑定的额外依赖。

## 12. LED 与 ILA 观察

板级 LED：

| 信号 | 含义 |
| --- | --- |
| `LED1` | `done`，程序完成标志。 |
| `LED2` | `verify_we`，输出 RAM 写使能。 |
| `LED3` 到 `LED7` | `cnt_test` 的部分位。 |
| `LED8` | `verify_RAM` debug 数据异或。 |

ILA probe：

| probe | 信号 |
| --- | --- |
| `probe0` | `test_vector_in[15:0]` |
| `probe1` | `verify_vector_out[15:0]` |
| `probe2` | `verify_we` |
| `probe3` | `verify_addr[4:0]` |
| `probe4` | `cnt_test[19:0]` |
| `probe5` | `done` |

首次 ILA 触发建议：

| 阶段 | 设置 |
| --- | --- |
| 第一轮 | 触发 `done == 1`，确认程序最终能结束。 |
| 第二轮 | 触发 `verify_we` 上升沿，确认 16 次写出过程。 |
| 第三轮 | 若输出异常，扩大采样窗口，同时观察 `verify_addr`、`cnt_test`、`verify_vector_out`。 |

首次上板重点确认：

1. `done` 最终拉高。
2. `verify_we` 出现 16 次有效写入。
3. 最后一次写地址为 15。
4. `cnt_test` 接近 157。
5. `verify_vector_out` 写入顺序与 `mem/FFT_output.coe` 一致。

## 13. 故障排查

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| Vivado 找不到 `xc7k325tffg900-2` | 未安装目标器件包或 license 不支持 | 在 Vivado installer/license manager 中补齐 Kintex-7 K325 FFG900 支持。 |
| `$readmemh` 找不到 `mem/*.mem` | 从错误工作目录启动 Vivado | 使用已更新脚本，或先 `cd` 到路线工程根目录。 |
| `.Xil/.../realtime` 无法删除 | Vivado 临时目录被锁 | 关闭 Vivado 进程，删除 `.Xil`，换新的 `OUT_DIR` 重跑。 |
| `done` 不拉高 | 复位、时钟、指令 ROM 或程序异常 | 检查 `KEY1` 极性、`CLK_50M`、`mem/instr_fft8.mem` 和 ILA 波形。 |
| `verify_we` 少于 16 次 | 程序未完整写出输出 | 回到仿真看 `verify_addr`、`cnt_test`、分支跳转。 |
| `WNS < 0` | 时序目标过高或关键路径过长 | 降频、换 strategy、比较 v7/v7b/v7c。 |

如果首轮上板失败，建议按下面顺序回退：

1. 先确认 Hardware Manager 能稳定连接 FPGA，不改 RTL。
2. 再确认复位和 `CLK_50M`，观察 `cnt_test` 是否变化。
3. 然后确认 Vivado log 中没有 ROM/RAM 初始化文件读取失败。
4. 再看 `done` 是否拉高，若不拉高回到仿真检查指令流。
5. 最后核对 `verify_we`、`verify_addr`、`verify_vector_out`，定位输出写回问题。
