# V26_scheduled_mul2_300

V26 基于 V22b 做纯汇编生成/寄存器分配优化，不修改 RTL，不新增指令，不修改 `cnt_test` 统计口径。

核心变化是把 `VERIFY_BASE` 的 `MOVI R5, #VERIFY_BASE` 从 Stage 3 输出前移到程序初始化区，也就是第一次输入读取之前。为了让 R5 在后续一直作为 verify_RAM 基址使用，原来用 R5 保存的临时虚部结果改用 R14。

## 结果

| 项目 | 结果 |
| --- | ---: |
| 回归 | 官方样例 + 20 组随机 PASS |
| `cnt_test` | 172 |
| MCU 频率 | 300 MHz |
| 推算耗时 | 0.573 us |
| WNS | +0.067 ns |
| TNS | 0.000 ns |
| WHS | +0.119 ns |
| THS | 0.000 ns |
| LUT | 1050 |
| FF | 675 |
| DSP | 0 |

## 结论

V26 是 V22b 之后第一条有效降周期且 300 MHz timing-clean 的路线。它把 `cnt_test` 从 173 降到 172，收益很小但规则风险低。由于 V22b 已完成上板验证，而 V26 尚未单独上板，当前建议把 V26 作为“最快候选”，V22b 仍作为“已上板验证主线”。
